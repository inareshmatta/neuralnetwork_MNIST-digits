## 🏗️ Model Architecture

```
DigitNet
├── Flatten              (batch, 1, 28, 28) → (batch, 784)
├── Linear(784 → 256)
├── BatchNorm1d(256)     normalizes activations within each batch
├── ReLU                 max(0, x) — only fire if positive
├── Dropout(0.3)         randomly drops 30% of neurons — prevents overfitting
│
├── Linear(256 → 128)
├── BatchNorm1d(128)
├── ReLU
├── Dropout(0.2)
│
├── Linear(128 → 64)
├── BatchNorm1d(64)
├── ReLU
│
└── Linear(64 → 10)      raw scores for each digit class
    ↓
    Softmax              converts scores to probabilities (sum to 1)
```

**Total parameters:** ~235,000 — trains in ~2 minutes on free Colab CPU.

## 🎨 Step-by-Step Breakdown

### Step 1 — Pixels to Numbers

Every image is just a grid of numbers. A 28×28 digit = **784 numbers** between -1 and 1 (after normalization). The notebook shows 5 sample digits side by side with their actual pixel values visible — exactly how the model receives input.

### Step 2 — What a Neuron Computes

One neuron takes all inputs, multiplies each by a learned weight, adds a bias, and applies ReLU:

```python
output = max(0, w₁·x₁ + w₂·x₂ + ... + w₇₈₄·x₇₈₄ + b)
```

The notebook draws this as a diagram with color-coded connections (green = positive weight, red = negative).

### Step 3 — Network Architecture

Visualizes the full 784→256→128→64→10 network as a node diagram with all layer connections, operation labels between layers, and neuron counts.

### Step 4 — Training

```python
for each batch:
    logits = model(images)               # forward pass
    loss   = CrossEntropy(logits, labels) # how wrong?
    loss.backward()                       # backpropagation
    optimizer.step()                      # update weights
```

Plots live training curves: loss, train accuracy, test accuracy per epoch.

### Step 5 — What Each Layer Sees

- **Activation distributions** per layer (how sparse is ReLU?)
- **Weight patterns** of layer 1 neurons reshaped to 28×28 — you can literally see what each neuron "looks for"
- **Output probabilities** as a bar chart

### Step 6 — Prediction Gallery

Shows 20 random test predictions with green (correct) or red (wrong) borders, confidence % per class, and top-3 guesses.

### Step 7 — Interactive Canvas 🎨

The draw pad — built with HTML5 Canvas + JavaScript. Features:
- Mouse and touch drawing support
- 28×28 pixel preview (what the network actually sees)
- Animated confidence bars for all 10 digits
- Layer activation visualizer (shows neurons firing in real time)

### Step 8 — Full Prediction Pipeline

Shows the complete journey of your drawing through the network — input image, predicted digit, confidence bars, and per-layer activity chart.

### Step 9 — Confusion Matrix

Finds which digit pairs the network finds hardest to distinguish and shows per-class accuracy. Typical findings: 4↔9 confused most often (both have loops), 3↔8 second most.

## 🙋 FAQ

**Q: Do I need to know maths?**
Basic school-level algebra is enough. Every formula is explained in plain English right next to the code.

**Q: Do I need a GPU?**
No. Trains in ~2 minutes on free Colab CPU. GPU makes it ~10x faster but isn't required.

**Q: Why does my drawing not predict correctly?**
A few reasons: MNIST digits are centered and thin — try drawing in the center of the canvas with a single clean stroke. Also the model was trained on 28×28 grayscale so thick, edge-touching drawings can confuse it.

**Q: What's the difference between this and just using a library like scikit-learn?**
We build and explain every component from scratch — the neuron math, backpropagation, activation functions. A library hides all of this. The goal here is understanding, not just accuracy.
